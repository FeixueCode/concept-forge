# Concept Forge 设计思路

## 1. 设计原点

这个 Skill 的核心不是把几个工具简单拼起来，而是建立一个稳定顺序：

```text
研究 -> 提炼 -> 选择载体 -> 生成成品 -> 检查能不能真的读、真的讲、真的交付
```

它吸收了三类能力：

- hv-analysis：负责研究、判断和横纵视角。
- Quarkdown：负责统一、可维护的知识库母稿。
- Slidev：负责把知识转成可讲解、可演示的结构。

最终产品形态是一个轻量知识生产流水线。

## 2. 为什么不是单纯回答

普通问答的缺点是：

- 答案停留在当前对话里。
- 搜过的概念下次还要重新搜。
- 信息散落，没有稳定格式。
- 看懂不等于记住，也不等于能讲给别人。

Concept Forge 把一次回答变成可复用资产：

- 短回答解决“我现在要马上懂”。
- HTML 解决“我之后能快速复习”。
- Quarkdown 解决“我需要一个统一知识库文本”。
- Slidev 解决“我需要把它讲出来”。
- 索引解决“我以后不要重复劳动”。

## 3. 核心抽象

### 3.1 Concept Record

每个概念先被整理成结构化记录。

它包含：

- 标题和别名。
- 一句直接解释。
- 一句话记住。
- 它是什么。
- 它为什么重要。
- 它怎么工作。
- 它不是什么。
- 相近概念对比。
- 关联概念地图。
- 例子。
- 记忆钩子。
- 小测验。
- 来源。

这个记录是所有输出的源头。

### 3.2 Quarkdown 母稿

Quarkdown 是正式阅读和长期维护的母稿。

它不追求花哨，而追求：

- 格式统一。
- 容易阅读。
- 容易扩展成报告。
- 可以作为 Slidev 的内容来源。

### 3.3 HTML 记忆面

HTML 是用户最常打开的学习页面。

它强调：

- 快速看懂。
- 卡片化。
- 可点击。
- 有小测。
- 能单独分享。

### 3.4 Slidev 讲解面

Slidev 是面向公开讲解的演示结构。

它强调：

- 开场一句话。
- 分步骤展开。
- 对比和例子。
- 现场小测。
- 三句话收束。

## 4. 难度分流

系统把问题分为三档：

### Quick

适合稳定、常见、低风险的小概念。

仍然生成所有产物，但研究更轻。

### Standard

默认档。

适合工具、方法、术语、概念对比和“是什么”类问题。

### Deep

适合复杂、变化快、有争议，或用户明确要求深度研究的问题。

会加强来源、历史脉络、横向对比和讲解结构。

## 4.1 交付分流

难度分流决定“研究多深”，交付分流决定“是否需要外部工具正式编译或导出”。

### 普通解释分支

用户只是想理解概念时，系统只生成源文件和 HTML，不要求安装 Quarkdown 或 Slidev。

典型说法：

- `X 是什么`
- `X 和 Y 有什么区别`
- `帮我理解这个概念`
- `这个词是什么意思`

### 正式阅读分支

用户想要报告、文档、PDF 或正式阅读材料时，系统进入 Quarkdown 编译链。

典型说法：

- `做成正式报告`
- `生成 PDF`
- `整理成长文档`
- `用 Quarkdown 编译`
- `我要可以交付的文档`

### 正式演示分支

用户想要演示、公开讲解、PPT、Slidev 或导出演示稿时，系统进入 Slidev 初始化和构建链。

典型说法：

- `做成演示稿`
- `公开讲解用`
- `生成 PPT`
- `生成 slides`
- `用 Slidev 预览/构建/导出`

### 双分支

如果用户同时要文档和演示，系统先以 Quarkdown 作为母稿，再从母稿提炼 Slidev 讲稿。

## 5. 复用逻辑

每次生成后，系统会更新 `knowledge/index.json`。

下次用户提问时：

1. 先查索引。
2. 匹配标题、别名、摘要。
3. 命中后直接返回已有答案和 HTML。
4. 只有在用户要求刷新、条目过期或概念变化快时才重做。

## 6. 可靠性设计

为了让 Skill 可以在普通环境里运行，基础能力只依赖 Python 标准库。

这意味着：

- 没有 Quarkdown 命令行工具，也能生成 `.qd` 母稿。
- 没有 Slidev 环境，也能生成 `slides.md`。
- 没有浏览器，也能生成自包含 HTML。
- 需要正式编译或导出时，再安装 Quarkdown 或 Slidev。

## 8. 未来扩展

后续可以自然加入：

- 更强的来源评分。
- 概念之间的图谱可视化。
- 批量导入浏览记录或聊天记录。
- Quarkdown 正式编译流水线。
- Slidev 静态站点发布。
- 多语言知识卡。
