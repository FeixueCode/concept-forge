# Concept Forge PRD

## 1. 产品定位

Concept Forge 是一个面向个人知识管理的轻量级 Codex Skill。

它解决的问题是：用户每天会遇到大量碎片化的新名词、新工具、新说法和新问题。普通搜索能快速得到信息，但信息常常零散、不可复用、难以记住。Concept Forge 把一次提问转成结构化知识资产，让它可以被阅读、被讲解、被复习、被索引和被再次调用。

一句话定位：

> 把一个问题锻造成一张能记住、能复用、能展示的知识卡。

## 2. 目标用户

- 经常接触新概念、新工具、新技术的人。
- 想建立个人知识库的人。
- 需要把复杂概念讲给别人听的人。
- 希望 AI 回答不只是一次性输出，而能长期积累的人。

## 3. 核心场景

### 场景 A：快速理解

用户问：“Harness 工程是什么？”

系统先给一句能立刻听懂的解释，再生成互动 HTML。

### 场景 B：概念对比

用户问：“Skill 和 Prompt 有什么区别？”

系统输出简明回答，并在知识页中补充相近概念、误区、例子和小测验。

### 场景 C：知识沉淀

用户第二次问同一个概念。

系统先查索引，命中后直接复用旧记录，避免重复研究。

### 场景 D：正式阅读或讲解

用户需要更完整的内容。

系统以 Quarkdown 作为母稿，以 Slidev 作为讲稿来源，同时保留互动 HTML。

## 4. 核心流程

```text
用户问题
  -> 规范化概念
  -> 查询已有知识库
  -> 判断难度
  -> 轻量研究
  -> 生成结构化记录
  -> 生成 Quarkdown 母稿
  -> 生成互动 HTML
  -> 生成 Slidev 源文件
  -> 校验
  -> 返回一句解释 + HTML
```

## 5. 功能需求

### 5.1 必须能力

- 识别用户问题中的核心概念。
- 支持别名、拼写错误和中英文混用。
- 每次回答先给 1 到 3 句直接解释。
- 每个概念生成结构化 JSON。
- 每个概念生成 Quarkdown 母稿。
- 每个概念生成互动 HTML。
- 每个概念生成 Slidev 演示稿源码。
- 维护本地索引，支持重复问题复用。
- 支持轻量、标准、深度三档研究。
- 输出中文为主，保留必要英文术语。

### 5.2 HTML 页面要求

- 自包含，直接打开即可阅读。
- 包含“它是什么、为什么重要、怎么工作、它不是什么”。
- 包含相近概念对比。
- 包含关联概念地图。
- 包含记忆钩子。
- 包含互动小测。
- 包含三句话带走。
- 包含来源和更新时间。

### 5.3 知识库要求

- `concept.json` 保存结构化信息。
- `concept.qd` 作为正式阅读母稿。
- `concept.md` 作为兼容备份。
- `sources.json` 保存来源。
- `index.json` 保存检索入口。

## 6. 非目标

- 不追求替代完整论文级 deep research。
- 不把所有问题都做成长报告。
- 不默认联网编译 Quarkdown 或 Slidev。
- 不要求用户每次都打开 HTML 才能得到答案。

## 7. 成功标准

- 用户提问后，能先获得一句清晰解释。
- 生成的 HTML 可以直接阅读和复习。
- 下次同类问题能命中已有条目。
- Quarkdown 和 Slidev 产物格式统一、可继续加工。
- Skill 可以复制到新环境后继续使用。
